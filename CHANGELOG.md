# 更新记录

## 0.2.0 - 2026-07-14

- 动态翻译调度升级：合并同一轮 DOM 更新，避免一次渲染被拆成大量串行请求；已翻译文本的样式变化不会重复请求。
- 修复原本隐藏的内容在展开后未翻译：覆盖 `details`、`aria-expanded`/受控区域、隐藏属性、CSS 状态切换、过渡和动画结束。
- 修复移动端工具栏状态丢失：展开、收起、贴边隐藏与恢复会立即串行保存；拖拽中断（`pointercancel`）也会保存最终状态。
- 设置升级为 `v2` 存储：自动迁移 `v1`，损坏值回退默认设置并备份，防止脚本初始化失败。
- 翻译请求更可靠：并发共享微软授权令牌，按总文本量拆分请求，缓存采用有上限的 LRU 策略。
- 修复高频动态页面的请求风暴：连续变更在 500ms 静默窗口后合并，所有微软翻译请求全局串行；遇到 `429`、超时或服务端故障会有限退避，并遵守服务端的 `Retry-After`。
- 翻译失败现在会显示 HTTP 状态、微软错误码或服务端说明，而非只在控制台输出泛化错误。
- 跳过所有可编辑区域（含 `plaintext-only`），并在动态节点移除后释放翻译记录。

## 0.1.5 - 2026-07-11

- 动态内容翻译现会监听 `class`、`style`、`hidden` 与 `aria-hidden` 的变化；Angular 等框架将已存在元素从隐藏状态切换为可见时，会重新扫描并翻译该元素范围。

## 0.1.4 - 2026-07-11

- 修复 Recraft 等在文档根节点使用 `translate="no"` 的网站无法执行全页翻译的问题；页面内部明确标记为不翻译的内容仍会跳过。

## 0.1.3 - 2026-07-11

- 修复链接后以逗号等标点开头的英文正文被误判为非正文、未参与翻译的问题。
- 工具栏新增当前网站自动翻译开关，可单独设为默认自动翻译或默认不自动翻译。
- 当前网站与全局自动翻译设置变更后，会立即翻译或恢复当前页面；工具栏悬浮提示会显示当前默认状态。
- 设置页移除自动翻译网站黑名单，已有黑名单会自动迁移为对应网站的“不自动翻译”偏好。

- 修复移动端收起工具栏后立即刷新或跳转页面时，收起状态可能未保存、再次展开的问题。

## 0.1.1 - 2026-07-11

- 在脚本管理器菜单新增“全页翻译”，用于启动整页翻译。
- 在脚本管理器菜单新增“翻译选中”，用于自动翻译当前选中的文本；未选中内容时显示提示。
