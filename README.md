# 网页翻译助手

`translator` 是一个基于 Microsoft Translator 的 Userscript 网页翻译工具。它尽量只替换文本节点，不重建页面 DOM，因此可用于普通网页和动态内容页面。

## 功能

- 翻译整页或当前可视区域。
- 支持划词翻译和输入文本翻译。
- 脚本管理器菜单提供“全页翻译”和“翻译选中”；后者会自动翻译当前选区。
- 保留原始文本，可一键恢复。
- 支持动态内容的增量翻译与内存缓存。
- 自动跳过代码、编辑器、公式、图标、隐藏内容、URL、邮箱和明确标记为不翻译的内容。
- 支持为当前网站单独设置默认自动翻译或默认不自动翻译，不影响手动翻译。
- 浮动工具栏支持展开、收起和拖到左右边缘后的贴边标签。
- 支持浅色、深色和触摸设备。

## 安装

1. 安装 Tampermonkey 或 Violentmonkey。
2. 下载并导入 [`dist/translator.user.js`](dist/translator.user.js)。
3. 打开任意普通网页，通过右下角工具栏或脚本管理器菜单开始翻译。

## 设置

可配置以下项目：

- 源语言与目标语言。
- 自动翻译页面。
- 当前网站自动翻译：通过工具栏的 `A` 按钮切换当前网站的自动翻译偏好；切换后立即翻译或恢复当前页面，悬浮按钮可查看当前状态。该偏好优先于全局默认值；修改全局“自动翻译页面”也会立即应用到当前页。
- 动态内容翻译、划词按钮和内存缓存。
- 批量大小与请求超时。

代码块、代码高亮容器、公式和在线编辑器始终跳过，避免误翻译技术内容。

## 工具栏

- 点击箭头可在展开与收起之间切换。
- 拖动工具栏可调整位置。
- 收起或展开状态拖到左右边缘时会吸附为贴边标签。
- 点击贴边标签会恢复到吸附前的展开或收起状态，并保持在同一侧边缘。

## 开发

```powershell
npm install
npm test
npm run build
```

- `npm test`：运行单元测试和 DOM 测试。
- `npm run build`：生成可安装的单文件脚本。
- `npm run dev`：监听 `src` 变更并重新构建。

## 说明

Microsoft Translator 的网页端点和授权令牌属于外部服务；服务策略或接口发生变化时，翻译可能失败，但脚本不会重建或破坏原页面结构。
